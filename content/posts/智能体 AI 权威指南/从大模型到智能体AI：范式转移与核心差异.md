# 从大模型到智能体 AI：范式转移与核心差异

## 核心观点

智能体 AI 的出现打破了传统 LLM "输入-输出"的范式。通过赋予 LLM **工具（手）、感知（眼）和记忆**，将博学的"思想家"转变为能与物理和数字世界互动的"实干家"。

## 关键差异：LLM vs 智能体

| 维度 | 传统 LLM | 智能体 AI |
|------|----------|-----------|
| **目标导向** | 概率预测下一个 Token | 目标达成为导向 |
| **状态管理** | 无状态（需手动传历史） | 持久化状态（长期记忆+任务状态+环境状态） |
| **推理模式** | 单次推理（A→B） | 主动循环（思考→行动→观察→...→结果） |

## 认知升级：系统 1 与系统 2

- **系统 1**：直觉思维，快速、无意识 → LLM 直接生成
- **系统 2**：逻辑思维，慢速、有意识 → 智能体的规划与反思（任务拆解→执行→自我修正）

智能体架构是在 LLM（强大的系统 1）外层，用工程手段包裹了一层系统 2 的逻辑外壳。

## 三大爆发要素

1. **推理能力临界点**：上下文学习与多步推演变得更可靠
2. **标准化交互协议**：Function Calling、MCP、A2A 等协议出现
3. **记忆与上下文扩展**：RAG 与更长上下文窗口

## 早期教训

- 幻觉传递：模型在工具参数中的幻觉会被实际执行
- 无限循环：缺乏最大步数限制导致死循环
- 上下文爆炸：Token 成本呈线性增长
- 错误处理缺失：工具返回错误时无法优雅降级

## 四大核心演进

1. **连接方式**：自定义 API → 标准化工具连接（动态发现）
2. **决策逻辑**：简单路由 → 认知架构（三思而后行，分层规划）
3. **安全控制**：Prompt 约束 → 独立策略层（零信任架构）
4. **协作模式**：单体多技能 → 多智能体 + 编排工作流（MAS）

---

> **延伸思考**：智能体正在重塑我们构建软件的方式——从"编写代码逻辑"转向"定义目标、提供上下文、设定约束"。代码形态正在经历从"思考载体"向"执行介质"的转变。
